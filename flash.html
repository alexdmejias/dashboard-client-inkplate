📱 Inkplate Dashboard Client Flasher

Flash the Inkplate Dashboard Client firmware to your Inkplate device using ESP Web Tools.

📦 Firmware Info: Loading...
ℹ️ Requirements:
  • Chrome, Edge, or Opera browser (WebSerial support required)
  • Inkplate 10 or Inkplate 10 v2 device
  • USB connection to your device
⚠️ First Time Setup:
  1. After flashing, create a config.txt file on the microSD card with your settings
  2. Use the example configuration from example-config.json as a template
  3. Insert the microSD card into your Inkplate device
  4. Power cycle the device to load the configuration

Configuration

The device can be configured via a config.txt file on the microSD card. Here's an example:

{
  "server": "https://your-server.com/image",
  "ssid": "your_wifi_ssid",
  "password": "your_wifi_password",
  "sleepTime": 300,
  "wifiTimeout": 30,
  "httpTimeout": 15,
  "showDebug": false,
  "wakeButtonPin": 36,
  "debugWindow": 10,
  "otaEnabled": false,
  "otaPort": 3232,
  "otaPassword": "",
  "otaTimeout": 30
}

OTA Updates (Over-The-Air)

After initial flashing, you can enable OTA updates in your config:

  1. Set "otaEnabled": true in your config.txt
  2. Optionally set an OTA password for security
  3. The device will wait for OTA updates for the configured timeout period after each wake
  4. Use PlatformIO to upload: platformio run --target upload --upload-port <IP_ADDRESS>
💡 Deep Sleep & OTA:

Since the device sleeps most of the time, OTA updates work best when:

  • The device is in debug mode (send "debug" command during the debug window)
  • You trigger an update during the OTA timeout window after each wake
  • You use the wake button to wake the device when you need to update
